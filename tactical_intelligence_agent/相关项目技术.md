## 1. 整体架构：两层分工

```
┌─────────────────────────────────────────────────────────────────┐
│  师兄 main — 公共控制面（Commander / Nacos / 协议 / BPEL）        │
│  · 工作流编排与 checkpoint                                       │
│  · Agent 发现、租约、心跳、宕机切换                               │
│  · 统一 A2A 接口与任务响应信封                                    │
│  · 附件 URI 协议（禁止 base64 直传）                              │
└───────────────────────────┬─────────────────────────────────────┘
                            │ HTTP: sendMessage / sendMessageStream
                            │ metadata: role=tactical_intelligence
┌───────────────────────────▼─────────────────────────────────────┐
│  你的 TIA — 业务实现层                                            │
│  tactical_intelligence_agent/  协议接入、载荷转换、产物上传         │
│  agent/                        感知→认知→通信 三技能流水线         │
│  config/default.yaml           RT-DETR 权重、geo、artifact 配置   │
└─────────────────────────────────────────────────────────────────┘
```

**原则**：宕机恢复、租约、failover 由 **Commander 框架**实现；TIA 只需 **注册、心跳、标准接口、幂等、统一 output**，即可自动享受这些能力。

## 3. 端到端运行流程

```
1. TIA 启动
   main.py → Nacos 注册(role=tactical_intelligence, status=idle) + 心跳线程
   → Uvicorn 监听 /health /ready /sendMessage ...

2. Commander 调度（BPEL invoke tactical_intelligence）
   lease_manager.acquire_one(role=tactical_intelligence)  → 实例标 busy
   → POST /sendMessage（Bearer JWT + payload）

3. TIA 协议层
   A2ABaseAgent.send_message
   → 检查 ready、work_item 缓存
   → execute_task(payload)

4. TIA 业务层
   commander_payload_to_batch → prepare_batch_for_inference
   → 感知→认知→通信 → SemanticIntelligencePacket
   → 可选 publish_processed_artifacts（标注图 URL）

5. 返回统一信封
   build_task_response(status=completed, output={intelligence_packet: ...})

6. Commander 落库
   apply_agent_result(role=tactical_intelligence)
   → context["intelligence_packet"] 追加条目
   → lease release(idle)

7. 若 TIA 宕机
   Commander 标记 unavailable → failover 到同 role 备用实例 → 同一 work_item 幂等
```

## 4. 分项讲解与代码

### 4.1 Nacos 注册与心跳

**师兄公共能力**：`NacosRegistry` 注册服务名 `A2A-Agent`，metadata 含 `role`、`status`；后台线程每 5s 刷新 `heartbeat_ts`，且**不会覆盖** Commander 写入的 `busy/unavailable`。

**TIA 接入**：启动时注册，与 recon/artillery 相同模式。

```16:43:tactical_intelligence_agent/main.py
if __name__ == "__main__":
    port = int(os.environ.get("TIA_PORT", os.environ.get("TACTICAL_INTELLIGENCE_AGENT_PORT", "8015")))
    heartbeat_interval = float(os.environ.get("A2A_HEARTBEAT_INTERVAL", "5"))
    role = os.environ.get("TIA_A2A_ROLE", "tactical_intelligence")

    agent = TacticalIntelligenceCommanderAgent(port=port, role=role)

    if os.environ.get("TIA_NACOS_REGISTER", "1") == "1":
        registry = NacosRegistry()
        ip = get_host_ip()
        registry.register_service(
            service_name=os.environ.get("TIA_NACOS_SERVICE", "A2A-Agent"),
            ip=ip,
            port=port,
            metadata={
                "role": role,
                "status": "idle",
            },
            heartbeat_interval=heartbeat_interval,
        )
    agent.start()
```

**讲解**：

- `role=tactical_intelligence`：Commander 按 role 查找实例。
- `status=idle`：可被租约领取；**不要**在业务代码里改回 idle，由 Commander `release()` 负责。
- `TIA_NACOS_REGISTER=0`：本地 HTTP 联调时可跳过 Nacos。

**环境变量**：

| 变量 | 默认 | 含义 |
|------|------|------|
| `TIA_PORT` | 8015 | HTTP 端口 |
| `A2A_HEARTBEAT_INTERVAL` | 5 | 心跳间隔（秒） |
| `TIA_A2A_ROLE` | tactical_intelligence | Nacos metadata.role |

### 4.2 继承 A2ABaseAgent — 标准 HTTP 与健康检查

**师兄公共能力**：所有业务 Agent 应继承 `A2ABaseAgent`，自动获得：

- `GET /health`、`GET /ready`、`POST /lifecycle/ready`、`GET /metrics`
- `GET /.well-known/agent-card`
- `POST /sendMessage`、`POST /sendMessageStream`
- `GET /workflows/{workflow_id}/work-list`

**TIA 接入**：只重写 `execute_task` 和 `execute_stream`，**不覆盖** `setup_routes`。

**基类 sendMessage 流程**（TIA 自动复用）：


**讲解**：

- `ready=false` 时返回 `AGENT_NOT_READY`，Commander 视为实例不可用并 failover。
- 同一 `work_item` 第二次请求直接返回缓存（`cached: true`），满足恢复/重试场景。
- TIA 的 `execute_task` 只负责业务；信封由基类 `build_task_response` 包装。

### 4.7 幂等：两层缓存

| 层级 | 位置 | 缓存内容 |
|------|------|----------|
| 协议层 | `A2ABaseAgent._task_response_cache` | 完整 HTTP 响应信封 |
| 业务层 | `TIA._result_cache` | `SemanticIntelligencePacket`（避免重复推理） |

```59:70:tactical_intelligence_agent/service.py
    def _get_or_process(self, payload: dict) -> SemanticIntelligencePacket:
        work_item = self._work_item_from_payload(payload)
        with self._result_lock:
            cached = self._result_cache.get(work_item)
        if cached is not None:
            return cached
        batch = commander_payload_to_batch(payload)
        packet = self._engine.process(batch)
        with self._result_lock:
            self._result_cache[work_item] = packet
        return packet
```

**讲解**：Commander 因超时/断心跳重派**同一 work_item** 时，TIA 直接返回已算好的情报包，不重复跑 GPU 推理。

## 10. 一句话总结

师兄 **main** 提供「多 Agent 操作系统」：发现、调度、租约、心跳、failover、标准协议、附件 URI、checkpoint。  
你的 **TIA** 在这个系统上注册为 `role=tactical_intelligence`，通过继承 `A2ABaseAgent` 接入 HTTP 与响应规范，通过 `payload_adapter` 接入输入规范，通过 `execute_task` 输出 `intelligence_packet`；**宕机恢复由 Commander 自动完成，TIA 只需活著、心跳、幂等、标准返回**。
