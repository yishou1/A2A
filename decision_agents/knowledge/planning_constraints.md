# Planning Constraint Knowledge

## PLAN-CONSTRAINT-001: Resource conservation

Tags: planning, resource, conserve

When resource conservation is requested, plans that use fewer available
resources should receive favorable explanation, provided they still cover the
highest-priority risks.

## PLAN-CONSTRAINT-002: Broad coverage preference

Tags: planning, coverage, broad

When broad coverage is requested, plans that cover all scheduled targets should
receive favorable explanation, even if they consume more resources.

## PLAN-CONSTRAINT-003: Compliance before handoff

Tags: planning, compliance, handoff

Every recommended plan must pass compliance and authorization checks before it
is handed to downstream demo modules.
