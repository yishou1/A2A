# A2A 工作流恢复与附件传递流程图

下面这张图概括了当前 A2A 项目中“工作流状态可恢复、任务可接管、附件以对象存储引用传递”的完整链路。

```mermaid
flowchart TD
    Start([启动 Commander / 接管请求])
    Args[读取 mode / workflow_id / resume / state_dir]
    Check{resume 且 checkpoint 存在?}
    NewCtx[创建新的 workflow context]
    LoadCtx[从 WorkflowStateStore 加载历史状态]
    Normalize[标准化 context / attachments]
    Save0[保存初始 checkpoint]
    Step{选择下一步}
    Build[构造任务信封
workflow_id / task_id / parent_task_id
input / context / attachments]
    Route[Local Runtime
或 Remote A2A]
    Agent[Recon / Artillery / Evaluator / Assault]
    Success{执行成功?}
    Apply[回写结果到 context
recon_report / strike_result / eval_score
commander_decision / assault_result]
    Pause[标记 paused
记录 last_error]
    Save1[保存 checkpoint]
    End([workflow completed])

    Start --> Args --> Check
    Check -- 否 --> NewCtx --> Normalize --> Save0 --> Step
    Check -- 是 --> LoadCtx --> Normalize --> Save0 --> Step

    Step -->|recon / artillery / evaluator / assault| Build
    Step -->|decision| Build

    Build --> Route --> Agent --> Success

    Success -- 是 --> Apply --> Save1 --> Step
    Success -- 否 --> Pause --> Save1 --> End

    Step -->|workflow_step 达到上限
或收到 RE-PLAN / ABORT| End
```

## 附件传递子流程

图片、视频等大文件不直接进任务体，只传对象存储引用和校验信息。

```mermaid
flowchart LR
    File[本地图片 / 视频文件]
    Upload[attachment_uploader]
    Hash[计算 sha256
推断 mime_type / kind]
    Store[上传到对象存储
或预签名 URL]
    Ref[生成 attachment ref
uri + checksum + meta]
    Task[写入任务信封
attachments 字段]
    Snapshot[写入 workflow context
与 checkpoint]
    Agent[Agent 读取引用
而不是二进制内容]

    File --> Upload --> Hash --> Store --> Ref --> Task --> Snapshot --> Agent
```

## 恢复接口入口

如果要外部接管某个工作流，可以通过恢复 API 直接按 workflow_id 续跑：

```mermaid
flowchart TD
    Client[外部 command / 运维入口]
    API[POST /workflows/{workflow_id}/resume]
    Store[WorkflowStateStore]
    Commander[CommanderAgent
resume=True]
    Continue[继续执行未完成步骤]

    Client --> API --> Store --> Commander --> Continue
```
